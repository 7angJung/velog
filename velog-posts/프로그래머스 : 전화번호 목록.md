<h3 id="💡-첫번째-시도-오답">💡 첫번째 시도 (오답)</h3>
<p>→ 배열 요소 중 <strong>가장 짧은 길이</strong>를 기준으로, 슬라이딩 윈도우처럼 모든 부분 문자열을 Map에 저장한다.<br />→ 동일한 부분 문자열이 2개 이상 등장하면, 접두어 관계가 존재한다고 간주한다.</p>
<pre><code class="language-java">import java.util.*;

class Solution {
    public boolean solution(String[] phone_book) {
        // 배열 요소 최소 길이 구하기
        int min_len = Integer.MAX_VALUE;
        for(String num : phone_book) min_len = Math.min(min_len, num.length());

        // 최소 길이로 나올 수 있는 모든 경우 맵에 저장
        Map&lt;String, Integer&gt; map = new HashMap&lt;&gt;();
        for(String num : phone_book) {
            for(int i=0; i&lt;=num.length()-min_len; i++) {
                String str = num.substring(i, i+min_len);
                map.put(str, map.getOrDefault(str,0)+1);
            }
        }

        // 1보다 큰 경우 접두어가 있는걸로 간주
        for(Map.Entry&lt;String, Integer&gt; entry : map.entrySet()) {
            if(entry.getValue() &gt; 1) return false;
        }

        return true;
    }
}</code></pre>
<h4 id="위-풀이의-결과">위 풀이의 결과</h4>
<ul>
<li>접두어 관계는 반드시 <strong>앞에서부터</strong> 시작해야 하는데, 중간 인덱스(<code>i</code>)부터 시작하는 부분 문자열까지 전부 비교하고 있어 잘못된 접근이다.</li>
<li>예를 들어 <code>&quot;123&quot;</code>과 <code>&quot;312&quot;</code>가 있다면, 슬라이딩으로 <code>&quot;23&quot;</code>, <code>&quot;31&quot;</code>, <code>&quot;12&quot;</code> 등이 겹쳐 접두어가 없는데도 false를 리턴하는 오탐이 발생한다.</li>
<li>히스토리 기준으로 <strong>0점, 66.7점</strong> 구간에 해당하는 풀이다.</li>
</ul>

<h3 id="💡-두번째-시도-오답">💡 두번째 시도 (오답)</h3>
<p>→ 슬라이딩이 문제였으니, <strong>시작 인덱스를 0으로 고정</strong>한다.<br />→ 각 번호의 앞 <code>min_len</code> 글자만 잘라서 Map에 저장하면, 최소 길이 번호가 다른 번호의 접두어인 경우를 잡을 수 있지 않을까?</p>
<pre><code class="language-java">import java.util.*;

class Solution {
    public boolean solution(String[] phone_book) {
        // 배열 요소 최소 길이 구하기
        int min_len = Integer.MAX_VALUE;
        for(String num : phone_book) min_len = Math.min(min_len, num.length());

        // 최소 길이로 나올 수 있는 모든 경우 맵에 저장
        Map&lt;String, Integer&gt; map = new HashMap&lt;&gt;();
        for(String num : phone_book) {
            String str = num.substring(0, min_len);
            map.put(str, map.getOrDefault(str,0)+1);
        }

        // 1보다 큰 경우 접두어가 있는걸로 간주
        for(Map.Entry&lt;String, Integer&gt; entry : map.entrySet()) {
            if(entry.getValue() &gt; 1) return false;
        }

        return true;
    }
}</code></pre>
<h4 id="위-풀이의-결과-1">위 풀이의 결과</h4>
<ul>
<li>최소 길이 번호가 접두어인 케이스는 어느 정도 잡히지만, 여전히 <strong>길이가 다른 번호들 사이의 접두어 관계</strong>를 전부 처리하지 못한다.</li>
<li>예를 들어 <code>&quot;12&quot;</code>와 <code>&quot;123&quot;</code>은 잡히지만, 접두어 관계 없이 앞 두 글자가 우연히 같은 번호들끼리 오탐이 생긴다.</li>
<li>히스토리 기준으로 <strong>54.2점, 79.2점, 91.7점</strong> 구간을 오가던 풀이의 원인이다.</li>
</ul>

<h3 id="💡-세번째-시도-정답">💡 세번째 시도 (정답)</h3>
<p>→ 발상을 바꾸자. 모든 번호를 Map에 넣어두고, 각 번호의 <strong>모든 접두어(prefix)</strong> 를 직접 잘라서 Map에 존재하는지 확인하자.<br />→ 어떤 번호의 접두어가 Map에 존재한다면, 그게 곧 다른 번호와의 접두어 관계를 의미한다.<br />(솔직히 이 방향은 AI의 도움을 받아서 풀었던 나의 과거 풀이를 참고했다...)</p>
<pre><code class="language-java">import java.util.*;

class Solution {
    public boolean solution(String[] phone_book) {
        Map&lt;String, Integer&gt; map = new HashMap&lt;&gt;();
        for(String num : phone_book) {
            map.put(num, map.getOrDefault(num,0)+1);
        }

        for(String num : phone_book) {
            for(int j=0; j&lt;num.length(); j++) {
                if(map.containsKey(num.substring(0,j))) {
                    return false;
                }
            }
        }

        return true;
    }
}</code></pre>
<h4 id="위-풀이의-결과-2">위 풀이의 결과</h4>
<ul>
<li>정확성과 효율성 테스트 드뎌 통과</li>
<li>히스토리상 <strong>2025-06-12, 2026-04-21</strong> 두 번 모두 이 방향으로 결국 통과했다.</li>
<li>다만 <code>Map&lt;String, Integer&gt;</code>을 쓰고 있는데, 이 풀이에서 Value(Integer)는 전혀 활용되지 않는다.</li>
<li>Key의 존재 여부만 확인하면 되므로, <code>Map</code>보다 <code>Set</code>이 더 적합한 자료구조다.</li>
</ul>

<h3 id="💡-네번째-시도-리팩토링">💡 네번째 시도 (리팩토링)</h3>
<p>→ <code>Map</code> 대신 <code>Set</code>을 사용해 불필요한 Value 관리를 제거하자.<br />→ 코드가 더 간결해지고, 의도도 명확해진다.</p>
<pre><code class="language-java">import java.util.*;

class Solution {
    public boolean solution(String[] phone_book) {
        Set&lt;String&gt; set = new HashSet&lt;&gt;();
        for(String num : phone_book) {
            set.add(num);
        }

        for(String num : phone_book) {
            for(int j=0; j&lt;num.length(); j++) {
                if(set.contains(num.substring(0,j))) {
                    return false;
                }
            }
        }

        return true;
    }
}</code></pre>
<h4 id="위-풀이의-결과-3">위 풀이의 결과</h4>
<ul>
<li><code>Map</code> → <code>Set</code>으로 교체하면서 코드가 훨씬 간결해졌다.</li>
<li><code>contains()</code> 하나로 접두어 존재 여부를 O(1)에 확인할 수 있어 가독성과 효율 모두 개선됐다.</li>
<li>최종 제출 겨우 통과~</li>
</ul>
