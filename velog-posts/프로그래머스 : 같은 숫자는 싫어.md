<h3 id="1트-정답-스택-활용">1트 (정답, 스택 활용)</h3>
<p>→ 스택의 <code>peek()</code>으로 직전 값을 확인하고, 현재 값과 같으면 건너뛴다.<br />→ 스택과 별도로 <code>List</code>에도 같이 담아서 최종 배열을 구성했다.</p>
<pre><code class="language-java">import java.util.*;
public class Solution {
    public int[] solution(int []arr) {
        Stack&lt;Integer&gt; stack = new Stack&lt;&gt;();
        List&lt;Integer&gt; list = new ArrayList&lt;&gt;();
        stack.push(arr[0]);
        list.add(arr[0]);

        for(int i=1; i&lt;arr.length; i++) {
            if(stack.peek()==arr[i]) continue;
            else {
                stack.push(arr[i]);
                list.add(arr[i]);
            }
        }
        int[] answer = new int[list.size()];
        for(int i=0; i&lt;list.size(); i++) {
            answer[i] = list.get(i);
        }

        return answer;
    }
}</code></pre>
<h4 id="위-풀이의-결과">위 풀이의 결과</h4>
<ul>
<li>정답은 맞지만, 스택과 리스트를 <strong>동시에 관리</strong>하는 구조가 불필요하게 중복이다.</li>
<li><code>Stack</code>만으로도 되는데 굳이 <code>List</code>를 따로 유지한 셈이다.</li>
<li>리팩토링 여지가 있어서 덱으로 다시 시도했다.</li>
</ul>
<hr />
<h3 id="2트-오답-덱-활용">2트 (오답, 덱 활용)</h3>
<p>→ <code>Stack</code> + <code>List</code> 조합 대신, 덱 하나로 처리하도록 변경했다.<br />→ <code>peekLast()</code>로 직전 값을 확인하고, 다르면 <code>offer()</code>로 뒤에 추가하는 방식.</p>
<pre><code class="language-java">import java.util.*;
public class Solution {
    public int[] solution(int []arr) {
        Deque&lt;Integer&gt; dq = new LinkedList&lt;&gt;();

        for(int i : arr) {
            if(dq.isEmpty()) {
                dq.offer(i);
                continue;
            }
            if(dq.peekLast() == i) continue;
            else dq.offer(i);
        }

        int[] answer = new int[dq.size()];
        for(int i=0; i&lt;dq.size(); i++) {
            answer[i] = dq.poll();
        }

        return answer;
    }
}</code></pre>
<h4 id="위-풀이의-결과-1">위 풀이의 결과</h4>
<ul>
<li><code>dq.size()</code>를 반복문 조건식에 직접 쓰면, <code>dq.poll()</code>로 요소를 꺼낼 때마다 <strong><code>size()</code>가 줄어들어</strong> 반복이 절반만 돌고 끝나버린다.</li>
<li>결과적으로 배열 뒷부분이 전부 0으로 채워지는 오답이 발생한다.</li>
</ul>

<h3 id="3트-정답-덱-활용">3트 (정답, 덱 활용)</h3>
<p>→ 반복문 진입 전에 <code>dq.size()</code>를 <code>size</code> 변수에 <strong>미리 저장</strong>해두고 고정값으로 사용했다.</p>
<pre><code class="language-java">import java.util.*;
public class Solution {
    public int[] solution(int []arr) {
        Deque&lt;Integer&gt; dq = new LinkedList&lt;&gt;();

        for(int i : arr) {
            if(dq.isEmpty()) {
                dq.offer(i);
                continue;
            }

            if(dq.peekLast() == i) continue;
            else dq.offer(i);
        }

        int[] answer = new int[dq.size()];
        int size = dq.size();
        for(int i=0; i&lt;size; i++) {
            answer[i] = dq.pollFirst();
        }

        return answer;
    }
}</code></pre>
<h4 id="위-풀이의-결과-2">위 풀이의 결과</h4>
<ul>
<li><code>size</code>를 미리 고정해두니 <code>poll()</code>로 꺼내도 반복 횟수가 줄어들지 않아 정상 동작한다.</li>
<li>덱 하나로 깔끔하게 처리 완</li>
</ul>

<h3 id="4트-리팩토링-스택덱-없이">4트 (리팩토링, 스택/덱 없이)</h3>
<p>→ 스택도, 덱도 필요 없다는 걸 깨달았다.<br />→ <code>arr[i] != arr[i-1]</code> 인덱스 비교만으로 직전 값을 확인할 수 있으니, 그냥 <code>List</code>에 바로 담으면 된다.</p>
<pre><code class="language-java">import java.util.*;
public class Solution {
    public int[] solution(int[] arr) {
        List&lt;Integer&gt; list = new ArrayList&lt;&gt;();
        list.add(arr[0]);
        for(int i=1; i&lt;arr.length; i++) {
            if(arr[i] != arr[i-1]) list.add(arr[i]);
        }

        int[] answer = new int[list.size()];
        for(int i=0; i&lt;list.size(); i++) answer[i] = list.get(i);
        return answer;
    }
}</code></pre>
<h4 id="위-풀이의-결과-3">위 풀이의 결과</h4>
<ul>
<li>스택/덱의 <code>peek()</code> 없이도 <strong>배열 인덱스로 직전 값 비교</strong>가 가능하다는 걸 활용했다.</li>
<li>자료구조 오버헤드가 사라지고 코드가 훨씬 간결해졌다.</li>
<li><code>isEmpty()</code> 체크도 필요 없고, <code>size</code> 저장 실수도 생길 여지가 없다.</li>
</ul>
